# 변경 로그

이 프로젝트의 주요 변경사항을 이 파일에 기록합니다.

형식은 [Keep a Changelog](https://keepachangelog.com/ko/1.0.0/)를 기반으로 하며,
이 프로젝트는 [Semantic Versioning](https://semver.org/lang/ko/)을 따릅니다.

## [미출시]

### 추가됨
- 백그라운드 서버 운영을 위한 데몬 모드
- 종합적인 문서 모음
- 모니터링 및 로그 스트리밍을 위한 REST API
- 실행 중인 데몬에 연결/분리 가능한 TUI 모니터
- 실시간 로그 스트리밍을 위한 Server-Sent Events (SSE)
- PID 파일 및 시그널 처리를 통한 프로세스 관리
- 크로스 플랫폼 지원 (Linux, macOS, Windows)

### 변경됨
- 서버가 이제 기본적으로 백그라운드 데몬으로 실행
- 더 나은 연결 상태 표시를 위한 TUI 개선
- 향상된 오류 처리 및 사용자 피드백
- 종합적인 가이드가 포함된 README 업데이트

### 기술적 개선
- syscall.Setsid를 사용한 적절한 데몬화 구현
- SIGTERM/SIGINT를 통한 정상 종료 처리 추가
- 별도의 데몬 관리 패키지로 모듈화된 아키텍처 생성
- 더 나은 신뢰성을 위한 Yamux 구성 개선
- 종합적인 로깅 및 모니터링 기능 추가

## [0.1.0] - 2024-01-XX

### 추가됨
- Tunnel Relay 초기 릴리스
- 마인크래프트 서버를 위한 역방향 터널링 기능
- 단일 TCP 연결을 통한 Yamux 멀티플렉싱
- 클라이언트 구성을 위한 기본 TUI
- Oracle Cloud 배포 지원
- 기본 로깅 및 상태 모니터링

### 기능
- 릴레이 서버로 아웃바운드 연결하는 호스트 클라이언트
- 플레이어 연결을 멀티플렉싱하는 릴레이 서버
- 실시간 연결 상태 표시
- 표준 마인크래프트 포트 (25565) 지원

## 개발 노트

### 버전 넘버링
이 프로젝트는 [Semantic Versioning](https://semver.org/lang/ko/)을 사용합니다:

- **MAJOR** 버전: 호환되지 않는 API 변경
- **MINOR** 버전: 하위 호환되는 기능 추가
- **PATCH** 버전: 하위 호환되는 버그 수정

### 변경 유형
- **추가됨**: 새로운 기능
- **변경됨**: 기존 기능 변경
- **더 이상 사용되지 않음**: 곧 제거될 기능
- **제거됨**: 제거된 기능
- **수정됨**: 버그 수정
- **보안**: 취약점 관련

### 변경 로그 기여
기여 시 다음을 지켜주세요:
1. `[미출시]` 섹션에 항목 추가
2. 변경사항에 현재 시제 사용 ("기능 추가" - "기능 추가됨" 아님)
3. 유사한 변경사항 그룹화
4. 해당하는 경우 이슈/PR 번호 참조

예시:
```
### 추가됨
- 새 기능 설명 (#123)

### 수정됨
- 버그 수정 설명 (#124)
```

## 마이그레이션 가이드

### 0.1.0에서 미출시 버전으로

#### 서버 사용법 변경
- **이전**: `./tunnel-server` (포그라운드에서 실행)
- **이후**: `./tunnel-server start` (데몬으로 실행)

#### 새로운 명령어
- `tunnel-server start` - 데몬 시작
- `tunnel-server stop` - 데몬 중지
- `tunnel-server status` - 상태 확인
- `tunnel-server monitor` - TUI 모니터 연결

#### 구성
- 모든 이전 명령줄 플래그 지원
- 새로운 데몬 관리 기능
- 모니터링을 위한 API 사용 가능

#### 클라이언트
- 클라이언트 사용법에 호환성 깨지는 변경 없음
- 더 나은 상태 표시를 위한 TUI 개선
- 향상된 오류 처리

### 호환성
- 0.1.0의 클라이언트는 미출시 버전의 서버와 호환
- 미출시 버전의 서버는 하위 호환성 유지
- API 엔드포인트는 새로운 추가 기능, 호환성 깨지는 변경 없음
